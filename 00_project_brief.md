# 项目 Brief：生活服务新人闯关训练

## 一句话定义

把新人第一周的产品体验任务，做成一套面向生活服务业务的游戏化闯关训练产品，帮助新人结构化理解抖音生活服务生态、消费决策链路、创作者路径和评价生产机制。

## 背景

leader 给到的新人作业是：

> 有空可以用户视角带入，了解下抖音的生态，创作者的路径，还有看评消费体验。

这不是一个普通报告任务。它真正要求新人完成三件事：

1. 从真实用户视角体验抖音生活服务。
2. 理解内容、交易、评价、创作者和商家之间的关系。
3. 把体感观察沉淀成产品判断。

如果只写一篇体验报告，产出容易一次性消耗。更好的做法是把它产品化为一套新人闯关机制，让你自己先跑通，再复用给后续新人。

## 目标用户

### 首批用户

- 新入职生活服务产品经理
- 新入职生活服务运营同学
- 评价、内容、交易、商家相关方向新人

### 第一位种子用户

- 你本人：抖音生活服务评价生产产品经理

## 核心目标

### 对新人

- 快速建立生活服务生态地图。
- 学会从用户目标出发做体验观察。
- 理解看评消费与评价生产的关系。
- 输出一份有结构、有证据、有机会点的产品体验报告。

### 对 leader / 导师

- 看到新人是否真正理解业务，而不只是泛泛体验。
- 通过关卡记录判断新人观察质量和产品归因能力。
- 复用同一套任务机制带下一批新人。

### 对团队

- 沉淀新人训练标准。
- 沉淀优秀观察样本库。
- 把新人 onboarding 从口口相传变成可配置流程。

## 成功标准

第一版成功，不以功能复杂度衡量，而看四个结果：

1. 你可以用它完成第一周产品体验报告。
2. 关卡任务足够清楚，不需要额外解释太多。
3. AI Game Master 能把体感观察追问成产品问题。
4. 产出的报告能被 leader 看出「有用户视角，也有产品判断」。

## 不做什么

- 不做真实数据抓取。
- 不自动爬取抖音内容。
- 不替新人生成虚假观察。
- 不做复杂积分商城、排行榜或社交系统。
- 不在第一版接入公司内部系统。
- 不把美团 PDE Skill 中的具体系统流程迁移过来。

## 可迁移的方法

来自历史 PDE Skill 的可迁移部分：

- Step 0 分诊：先确认用户处于哪个阶段。
- 分阶段推进：一次只聚焦当前阶段。
- 固定项目信息：明确主题、用户、产出和约束。
- 确认点：关键动作前让用户确认。
- 汇报卡：每个阶段结束有结构化状态。
- 已知坑点：提前暴露容易跑偏的地方。
- 边界：明确系统不替用户做什么。

## 项目阶段

| 阶段 | 名称 | 目标 |
|---|---|---|
| A | 项目定义 | 明确用户、场景、交付物、边界 |
| B | 关卡设计 | 设计 6 个标准关卡和 Boss 关 |
| C | AI Game Master | 设计任务引导、追问、评分、报告生成 |
| D | 自己试跑 | 你作为第一个新人完成一轮 |
| E | 报告生成 | 输出第一版体验报告 |
| F | 团队复用 | 改造成其他新人可用的模板 |

## 第一版推荐形态

先不急着做完整软件。第一版采用「文档 + 表单 + AI 对话」即可：

- 文档承载关卡规则。
- 表单或 Markdown 承载提交记录。
- AI Game Master 负责追问、评分和报告生成。

如果试跑证明有价值，再升级为飞书应用、妙搭应用或 Web 工具。
